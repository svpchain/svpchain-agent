svpchain agent — macOS 运行前先阅读
========================================

本安装镜像（DMG）包含：

  • svpchain agent.app   图形化配置工具（导入密钥、生成 MCP 配置）
  • 本说明文件
  • READ-BEFORE-RUN.txt       英文版说明（English guide）

.app 内已附带 svpchain-mcp 签名程序，无需单独下载。


一、安装步骤
------------

1. 双击打开 DMG 文件（例如从「下载」文件夹）。
2. 将 svpchain agent.app 拖入「应用程序」文件夹（或 DMG 窗口中的 Applications 快捷方式）。
3. 按下面「首次打开」方式启动一次。
4. 在应用内完成：导入密钥 → 生成 MCP 配置 → 复制到 Cursor 等客户端设置。


二、首次打开（重要）
--------------------

本应用未使用 Apple Developer ID 公证（notarization）。从浏览器下载后，
macOS 可能拦截直接双击，这是正常现象，不是病毒。

【常见弹窗 — macOS Ventura 及更新版本】

  标题类似：「"svpchain agent.app" Not Opened」/「"svpchain agent.app" 未打开」
  正文：Apple could not verify … is free of malware …
  按钮只有「Move to Trash / 移到废纸篓」和「Done / 完成」——没有「打开」。

  请先点「完成 / Done」（不要点「移到废纸篓」），再按下面方式 A 或 B 启动。

请任选以下一种方式首次启动：

【方式 A — 推荐】右键打开

  1. 在 Finder 中找到 svpchain agent.app
  2. 按住 Control 键并点击（或双指点击触控板）
  3. 选择「打开」
  4. 在第二个弹窗中再次点击「打开」
  5. 之后同一台 Mac 上通常可直接双击打开

【方式 B】系统设置中「仍要打开」

  1. 先双击一次应用（出现上述「未打开」弹窗后点「完成」）
  2. 打开「系统设置」→「隐私与安全性」
  3. 向下滚动，在「安全性」区域找到 svpchain agent 相关提示
  4. 点击「仍要打开 / Open Anyway」
  5. 在确认框中再次确认打开

【方式 C】去掉下载隔离标记（从 GitHub 下载后仍无法打开时）

  在终端执行（将路径改为实际解压位置）：

  xattr -cr "/路径/svpchain agent.app"

  然后仍建议使用「右键 → 打开」完成首次启动。

旧版 macOS 也可能出现「无法打开」或「来自身份不明的开发者」，处理方式同上（方式 A 或 B）。


三、钥匙串访问提示
------------------

首次导入签名密钥时，macOS 可能弹出「钥匙串访问」授权对话框。
请选择「允许」或「始终允许」，以便应用读写本地签名密钥。
密钥保存在系统钥匙串中，不会写入普通文本配置文件。


四、配置完成后
--------------

GUI 仅用于一次性设置。日常使用时，Cursor 等 MCP 客户端会自动
通过 stdio 启动 svpchain-mcp 签名服务，无需保持 GUI 常开。

详细说明见项目 README：
https://github.com/svpchain/svpchain-agent


五、仍无法打开？
--------------

  • 确认 macOS 版本 ≥ 11.0
  • 确认下载完整且校验 Release 页的 SHA256SUMS（如有提供）
  • 仅从官方 GitHub Release 下载，勿使用来路不明的重打包版本

如有问题，请在项目 Issues 反馈并说明 macOS 版本与具体报错截图。
